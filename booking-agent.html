// overview

A complete appointment booking agent built on Make.com. Instead of manually checking a calendar and replying to every booking request by hand, this agent handles the entire flow the moment someone submits the Tally form.

The agent checks the requested date and time against existing Google Calendar events. If the slot is free, it books the event, logs the confirmed appointment to Google Sheets, and sends a confirmation email automatically. If the slot is already taken, it pulls the conflicting events, hands that context to Google Gemini AI, and lets the AI write a natural response — logging that attempt separately and emailing the user back.

Zero manual scheduling. Zero double-bookings. 24/7.

// build story — the date formatting bug

The workflow logic worked on the first pass. What didn't work — for a while — was something much smaller and much more annoying: getting a date to display the same clean way in every place it showed up.

First symptom

Clean date in Make, raw ISO in Google Sheets

The Set Variable step showed a perfectly readable date — "July 24, 2026 1:00 PM" — in Make's own execution log. But the Google Sheets row it fed into showed a raw ISO timestamp like 2026-07-12T02:00:00.000Z instead.

First wrong turn

Column formatting and value input option

The first suspects were Google Sheets column formatting and Make's "value input option" (RAW vs USER_ENTERED). Both were checked and corrected — column set to plain text, input option set to RAW — but the ISO string kept showing up.

Root cause found

The date was never text to begin with

The readable date in Make's log was only a display convenience. Internally, date+time stayed a Date object until something explicitly converted it. Any field it landed in without an explicit format call serialized it back to ISO 8601 — regardless of Sheets settings.

Second bug, same root cause

Broken concatenation in the Gmail body

The confirmation email had a second, unrelated-looking symptom — a garbled string like "AM0iam/7amni7/31/2026am" tacked onto the end of an otherwise correct sentence. Tracing it back showed a leftover broken date expression concatenated alongside a working one, left over from earlier edits.

Fixed

formatDate() applied at every output point

The fix was the same in both places: wrap date+time in Make's formatDate() function wherever it left the pipeline as plain text — once in the Google Sheets mapping, once in the Gmail body — and delete the stray leftover expression. Both the sheet log and the confirmation email now show a clean, consistent date.
