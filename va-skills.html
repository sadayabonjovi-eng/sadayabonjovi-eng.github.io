System · gmail-management · live

Gmail Inbox Management System

A complete, operational email management system built inside Gmail using only its native tools — no third-party apps required. Designed around a real client delegation scenario, the system transforms a 200+ email inbox into a clean, prioritized, and fully actionable workspace that a VA can manage without ever knowing the client's password.

The foundation is a custom label architecture — seven top-level categories (CLIENT, URGENT, FOLLOW UP, TO DO, TO READ, WAITING FOR REPLY, COMPLETED) each broken into nested sub-labels like CLIENT/Active, URGENT/High, and FOLLOW UP/Pending Payment — so every email is tagged by both category and urgency at a glance. Every label is color-coded for instant visual scanning.

On top of labels, a four-star priority system adds a second visibility layer — yellow for needs attention, red bang for urgent, green check for done, blue info for waiting on a reply. The inbox itself is split into multiple sections so client emails always surface first, above promotions and low-priority reads. Auto-filters sort incoming mail on arrival, and five canned response templates — acknowledgement, follow-up, schedule call, payment reminder, and task completed — mean common replies go out in seconds. Schedule send and snooze handle timing, and a professional email signature goes out on every message automatically.

Access is managed through Gmail delegation — the VA operates the inbox from their own Google account, the client stays in full control, and access can be revoked instantly at any time.

System · calendar-management · live

Google Calendar Management System

A complete, operational calendar management system built inside Google Calendar using only its native tools — no third-party apps required. Designed around a real two-client VA scenario, the system organizes meetings, deadlines, recurring tasks, and out-of-office blocks into a clean, color-coded workspace that scales to any number of clients without confusion.

Each client gets a dedicated color-coded calendar — blue for Client 1, purple for Client 2 — so every event is instantly identifiable by client without opening it. On top of calendar color, an emoji prefix system adds a second layer of meaning: 📞 for calls, ⚡ for urgent, 🚫 for out-of-office blocks, 💰 for payment deadlines, and more — so event type and urgency are readable at a glance.

The system runs on three recurrence patterns — daily, weekly, and bi-weekly — covering standups, check-ins, strategy meetings, weekly reports, and monthly reviews. All meetings include video conferencing links embedded directly in the event description. Access is managed through Google Calendar sharing — the VA manages events from their own account, the client stays in full control, and access can be revoked instantly at any time.

System · google-drive · live

Google Drive Management System

A fully structured, permission-controlled, and documented Google Drive system built for a simulated small business client — Sarah's Bakery — covering everything a VA handles on day one and ongoing: building a clean folder structure from scratch, setting granular permissions for each team member, recovering accidentally deleted content, organizing files for fast retrieval, and maintaining a template system for recurring documents.

The Drive runs on a two-layer folder system — utility folders prefixed 00A/B/C (📥 INBOX, 📋 README, 📁 TEMPLATES) float at the top, while numbered content folders (01–05) hold the actual client work. Every folder is color-coded by category and numbered to control sort order, with Archive and Templates subfolders inside each one. A dedicated Marketing Plan folder groups monthly plans by year, so the structure stays clean as the business grows.

Access is managed per role — the accountant gets Viewer-only access to Reports, the social media manager gets Editor access to Marketing — with resharing disabled for both. Sensitive folders use folder-level privacy lock so parent-folder access doesn't cascade into confidential files. Version History with named checkpoints handles accidental edits, and a master template system ensures monthly recurring documents are always produced from a clean original.
